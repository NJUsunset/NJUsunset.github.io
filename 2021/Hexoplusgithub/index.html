Posted on 2021-11-15 Edited on 2021-12-25 In SHARE

Hexo + GitHub to set up your personal blog

To be the same with most blogs set up on hexo, I would like to record main steps when I deal with my blog, and I hope this simple record could help you to some degree if you want to set up your own static blog on hexo.

Before the record, I list some pages that could be helpful (And did help me when I start to establish this blog).

  • Documentation|Hexo
  • GitHub+Hexo搭建个人网站详细教程(知乎)
  • NexT - Theme for Hexo
  • next theme GitHub page(GitHub)
  • GitHub Pages docs(GitHub docs)

OK, let’s start.

What are we going to do?

We will use hexo to generate a blog website and display it with github pages. Just like the one on which you read this post.

What is Hexo?

Hexo is a fast blog framework. You can write post in Markdown or other markup languages and Hexo generates static files constructing a static blog with beautiful themes.

What is GitHub and GitHub Pages?

The world largest HOMOSEXUAL DATING SITE.

Ok, let’s be more serious. GitHub is the largest code hosting platform of the world. Over 65 million developers “shapes the future of software”, together. You can contribute to the open source community, manage your git repositories, review code like a pro, track bugs and features and power your CI/CD and DevOps workflow.

The main service we are going to use is called GitHub Pages. It provides us a simple way of directly constructing public websites through your repositories on github.com. We can use GitHub Pages to showcase some open source projects, host a blog, or even share your resume. And here we will use it to publish our personal plog website produced by hexo.

Preparation beforehand

Local side

To use hexo on our computer (and push our posts to github), we need to install Node.js and Git on our computer. Below I display one way each for Windows and Linux to install them. You can skip this part if you are familiar with them.

For Windows system

To install Node.js on windows system, just need to go to the official website of Node.js and download .msi package. (Currently node-v16.13.0-x86.msi as the newest) After download finished, double click .msi file and follow the lead to install Node.js.

Git installation is mostly same as what we do above. Visit git download page for windows to download .exe package. And run it after download complete to install git

After installation complete, press Win+R, input cmd and then press Enter to run CommandPrompt. Type node -v, npm -v and git --version and press Enter each to check. You would receive three build number if everything is right.

For Linux system

Git is included in package manager on most linux distribution. You can directly install with command like sudo apt install git or something alike. But you still have to download nodejs package for linux versions.
First, go to /usr/local/.

Then, use wget to download .tar.gz package to where you want to store it. You may need to install it first with sudo apt install wget.

Or mirror site for faster download speed.

And use command tar xzvf <package-name> to decompress it. (add sudo before command if warned Permission Denied)

Then create symbolic link.

Then input node -v to verify, you would get a build number if everything is fine.

Remote side

The necessity in remote side is a github account. We just need functions of free account unless you don’t want to publicize your website code. And we can also apply(buy) a domain name so that we can visit our blog site through custom address. (e.g. You can visit my blog through https://blog.wkfspace.xyz other than https://njusunset.github.io)

It would be a small piece of cake to register a github account. But we would have some trouble setting domain name. We need to add a record in type ‘CNAME‘ towards ‘.github.io‘. And later we will set in github to make it work.

Transfer blog to github pages

After successfully run the blog on our devices, we need to deploy it to github pages so that others can visit the blog through the internet. Here we’ll use a hexo plugin called hexo-deployer-git to fulfill it.

As github has stopped support to login in with name and password from remote devices, we have to create a token (a kind of temporary password). Login to the github website and open setting page. Then find the “Developer settings” section, click in and then click “Personal access tokens”. Click “Generate new tokens”. In this case we only need permissions in “repo” part so click it on and choose a proper expiration time (To avoid further confusion in the future, I chose no expiration time. But mostly it wasn’t recommended for secure consideration). After finish setting (including a tag for the token) click generate and we will get a long serial, click the copy button next to it and save it somewhere safe. Warning: You won’t see it after you leave that page. So if you forget it you can only regenerate a new one to replace.

If you have applied your personal domain name and you want to visit the blog through it or subdomain like blog.wkfspace.xyz. First, make sure you have add proper CNAME record on your domain manage page. The create a file named “CNAME” in “source” folder in the blog folder, and then open it with txt editor and wirte your domain name in it. Attention that the file has no extensions, not CNAME.txt or CNAME.md, just CNAME. You can fulfill it on windows by creating a CNAME.txt and deleting the extension .txt later. And the make a construct and push to github.

Then github will do a check to connect to your domain, you can check the process in setting “pages” section. After success, your can visit your blog through your own domain name.

Personalize

Most setting can be modified in _config.yml file. You can change themes, icons, classifications and many other aspects of your blog. Please refer to official documentation for more info.

Plus, you can visit Plugins | Hexo to find all kinds of plugins offering more functions for your blog.
